# ワイヤーフレーム（drawio）フォーマット定義

対象: ドキュメントルート配下で `*.drawio` を含むディレクトリ内のファイル（通常 `screens/`）

## ファイル命名規則

```
<画面名>.drawio
例:
  admin-login.drawio
  admin-dashboard.drawio
  user-profile.drawio
```

## 生成ルール

- テンプレートはこのスキルの `assets/drawio-templates.md` を Read して使う
- 画面定義（`.mdx`）の操作フローに記載された要素をすべて含める
- レイアウトは既存の drawio ファイルを参考に、画面の種類に応じて選択する

## レイアウト選択

既存の drawio ファイルを Read してレイアウトパターンを把握し、同じスタイルを踏襲する。
代表的なレイアウトタイプ：

- **sidebar** タイプ: サイドバー + ヘッダー + メインコンテンツの3カラム構成
  - メニュー項目は既存の drawio ファイルから読み取る
- **centered** タイプ: ヘッダー + センタリングされたカード
- **fullwidth** タイプ: ヘッダー + 全幅コンテンツ

## サイズ基準

デフォルト値（既存ファイルに合わせて調整）：

| 要素 | サイズ |
|---|---|
| 画面全体 | 幅 1200px、高さは内容に応じて調整 |
| ボタン | 幅 120px、高さ 36px |
| テーブルヘッダ行 | 高さ 40px |
| テーブルデータ行 | 高さ 36px |
| 入力フィールド | 高さ 36px |
| カード余白 | 16px |

## スタイル規則

デフォルト値（既存ファイルに合わせて調整）：

| 項目 | 値 |
|---|---|
| フォント | Helvetica |
| アクセントカラー | `#4A90D9`（青）のみ |
| 基調 | グレースケール |
| ID命名 | `cell-<連番>` （状態バリエーションには `-bg`, `-active` 等のサフィックスを付与） |
| ラベル | 日本語 |

## ID 番号帯の割り当て

| 番号帯 | 用途 |
|---|---|
| 2〜9 | ブラウザ枠 |
| 10〜19 | ヘッダー |
| 20〜29 | サイドバー |
| 30〜39 | メインコンテンツ枠 |
| 40〜 | 画面固有のコンポーネント |

## ドキュメントへの埋め込み

埋め込み構文は画面定義側で一元管理する。`references/screen-doc.md` の「必須 5 セクション構成」を参照し、`## ワイヤーフレーム` セクションの `<Drawio content={wireframe} />` 埋め込みに従う。

ドキュメントが `.md` の場合は `.mdx` に変更する（drawio インポートが必要なため）。

## 品質基準

- app.diagrams.net で正常に開ける有効な XML であること
- UIコンポーネントが重ならず整然と配置されていること
- 過度な装飾をしないこと（ワイヤーフレームとしての視認性を保つ）
